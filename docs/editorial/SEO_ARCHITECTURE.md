# SEO Architecture: Pilar + Tool

## A) Modelo Pilar + Tool

- El **artículo pilar** explica el concepto, contexto, casos y decisiones.
- La **tool/calculadora** ejecuta el cálculo o simulación.
- Pilar y tool deben tener **intenciones distintas** y no competir por la misma keyword primaria.

Regla operativa:

- Pilar: intención informacional (`qué es`, `cómo funciona`, `guía`, `paso a paso`).
- Tool: intención transaccional/funcional (`calculadora`, `simulador`, `conversor`, `estimar`).

## B) Regla de titles

- No repetir el mismo núcleo semántico en dos URLs para la misma intención.
- Ejemplo prohibido: repetir `Cómo calcular sueldo líquido en Chile` en dos rutas.
- Titles de tools deben ser funcionales, no educativos.

Formato sugerido:

- Pilar: `Tema en Chile: qué es, cómo funciona y ...`
- Tool: `Calculadora/Simulador de [resultado] en Chile`

## C) Interlinking mínimo obligatorio

- Cada pilar enlaza al menos 1 tool del mismo cluster.
- Cada tool enlaza al menos 1 pilar del mismo cluster.
- Anchor text explícito, orientado a intención.

Anchors válidos:

- `Usa la calculadora de sueldo líquido`
- `Revisa la guía completa de renegociación Superir`

Anchors no válidos:

- `haz clic aquí`
- `más info`

## Reglas anti-canibalización (enforcement)

- Definir keyword primaria por URL antes de publicar.
- Si dos piezas compiten por la misma intención:
  - fusionar contenidos, o
  - reposicionar una URL a intención secundaria, o
  - usar redirección/canonical según corresponda.
- Ninguna tool debe duplicar la narrativa completa del pilar.
- Ningún pilar debe sustituir la ejecución funcional de la tool.
